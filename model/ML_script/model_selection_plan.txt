疫情 next-day 预测任务：模型选择结论与实施方案（先不写代码）

一、任务特点（决定模型选择）
1) 数据形态：
- 你们当前是 country-date 面板数据（结构化表格特征），不是图像、也不是纯文本。
- 核心目标是回归任务：预测各国家下一天新增病例（target_next_day_new_cases）。

2) 业务要求：
- 需要可解释、可快速迭代、能稳定上线。
- 后续还有“点击首发国家（Patient Zero）”的交互推理需求。

二、结论（优先级）
结论：第一阶段最推荐“传统机器学习中的梯度提升树模型”（LightGBM / XGBoost / CatBoost），不建议一开始就上 CNN 或 Transformer。

推荐顺序：
- P0：基线规则模型（naive baseline）
- P1：梯度提升树（主力首选）
- P2：时序深度模型（TFT / LSTM / Transformer）作为对照增强
- P3：跨国传播建模（图结构或传播仿真层）

三、为什么不是 CNN？
CNN更适合图像、网格、局部空间卷积场景；你们的数据是国家-日期结构化特征表，直接用CNN通常不如树模型高效。

四、为什么暂不优先 Transformer？
Transformer在超大规模长序列上有优势，但当前任务是“结构化特征 + 单步 next-day 预测”，且需要快速验证与可解释性：
- 训练成本高、调参复杂。
- 业务早期不一定显著优于树模型。
- 更适合作为第二阶段性能冲刺模型，而不是 MVP 首选。

五、为什么首选梯度提升树（GBDT）？
1) 对表格数据非常强：
- 能处理非线性关系、特征交互、缺失值与不同量纲特征。

2) 样本效率高：
- 在中等规模数据上通常优于深度模型冷启动表现。

3) 可解释性强：
- 可通过特征重要性、SHAP 分析解释“为什么某天某国预测高”。

4) 迭代速度快：
- 训练和调参效率高，适合你们当前项目节奏。

六、建议的分阶段方案
阶段A：建立最低可行基线（必须先做）
- Baseline-1：y_hat(t+1) = new_cases(t)
- Baseline-2：y_hat(t+1) = 近7天均值
意义：保证后续复杂模型“确实比简单规则更好”。

阶段B：主力模型（首选）
- 模型：LightGBM 回归（可并行对比 XGBoost / CatBoost）
- 输入：你们已清洗的特征 + 滞后特征 + 政策/疫苗/检测/静态特征
- 目标：target_next_day_new_cases
- 输出：全国家下一天新增病例

阶段C：时序深度模型对照（可选增强）
- 候选：Temporal Fusion Transformer（TFT）或 LSTM encoder-decoder
- 用途：验证是否在“峰值段、突变段”有明显提升
- 规则：必须在同一时间切分下与GBDT公平比较

阶段D：Patient Zero 交互扩展（后续）
- 不是只靠单一预测模型就能完美实现，需要“预测模型 + 传播机制层”
- 可选方案：
  a) 在特征中加入跨国传播强度（邻国、区域、航线、国际流动代理特征）
  b) 或增加图模型/GNN做跨国关联建模
  c) 推理时做滚动预测并注入首发国冲击

七、评估与上线建议
1) 评估指标：
- MAE、RMSE、SMAPE（至少三项）
- 分国家评估（不能只看全局平均）
- 分阶段评估（平稳期 vs 爆发期）

2) 验证方式：
- 严格时间切分（你们已做）
- 建议加 rolling window 回测

3) 目标判断标准：
- 新模型必须稳定优于 Baseline
- 在主要国家和主要时间段不出现系统性偏差

八、最终建议（可直接执行）
当前项目最优实践：
- 先用 LightGBM 做主模型，快速拿到稳定、可解释、可部署结果；
- 再用 Transformer/TFT 做性能冲刺对照；
- Patient Zero 的最终效果通过“模型 + 传播机制层”联合实现，而不是单一网络结构硬做。
